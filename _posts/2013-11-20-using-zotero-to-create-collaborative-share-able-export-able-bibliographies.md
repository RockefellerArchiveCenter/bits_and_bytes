Collect references once, export and share infinitely. That's what tools like <a title="Endnote" href="http://endnote.com" target="_blank">Endnote </a>and <a title="Zotero" href="http://zotero.org" target="_blank">Zotero </a>come down to. In grad school I used <a title="Endnote" href="http://endnote.com/" target="_blank">Endnote </a>to manage my references and then cite them as needed in Word documents (papers, my dissertation...). The software would automatically create bibliographies from cited material, in virtually any format imaginable (MLA and Chicago are just the beginning). But I never took it a step further to create an online, shareable library. It wasn't until I began work on a Ford Foundation bibliography here at the Archive Center that I saw the benefits of an online reference-storing tool. Thanks to Hillel's suggestion, I began to build one using <a title="Zotero" href="http://www.zotero.org/" target="_blank">Zotero</a>, a free program .

Although I'm only about a month into using the software, I thought I'd outline the process for creating online bibliographies in Zotero, while adding some comments about RAC-specific uses and parts I still need to work out. I'm interested in discussing the "why" as much as the "how," so please comment or feel free to chat about this with me, if you think it may be something you could use, too!

**Why use Zotero?**

My use of Zotero is to answer several needs: For one, researchers working on the Ford Foundation collection at the Archive Center will undoubtedly be interested in what has already been written on the foundation, since more open access to their archives is only a recent development and research isn't as established as it has been for other foundations. For example, some researchers are familiar with other RAC collections but new to Ford, so would benefit from some of the institutional histories found in secondary materials.

Our own internal uses are numerous: knowing where to find information about a program or time period can help us answer research requests, or build a foundation for our Research & Education projects. Having the bibliographical information in an online format means that many of us can contribute to it, and that it can be updated constantly and immediately available. Importantly, we can create a bibliography to share as a pdf or on paper for those external to the Archive Center. The idea is to create a working list of relevant secondary material that is up-to-date and quickly accessible.

**How?**

****The Zotero site has an <a title="Zotero Quick Start Guide" href="http://www.zotero.org/support/quick_start_guide" target="_blank">excellent tutorial</a> on getting started. But in this post I'll take you through an example: first adding a reference, then using the Zotero Standalone program to create a recognizable, useful bibliography.

[<img class="size-full wp-image-879 alignleft" style="margin: 5px;border: 1px solid black" alt="ZoteroAddRecord" src="http://rockarch.org/programs/digital/bitsandbytes/wp-content/uploads/2013/11/ZoteroAddRecord.png" width="156" height="115" />](http://rockarch.org/programs/digital/bitsandbytes/wp-content/uploads/2013/11/ZoteroAddRecord.png)**Adding a New Reference**

{Note that there are even more immediate ways of adding a reference from an online source using <a title="Zotero Firefox extension" href="http://download.zotero.org/extension/zotero-4.0.15.xpi" target="_blank">an extension for the Firefox browser</a>.}

This example is going to be a walk-through of what I did when I came across a journal article I wanted to add to our Ford Foundation bibliography. After logging in to Zotero, and then choosing the "My LIbrary" tab at the top, I clicked on the green "add new" icon (pictured above).

I then chose the item type from the drop-down menu and filled in the fields for which I had information (title, author, publication, etc.). This will be the only time I'll need to type in this information. It will be exported later into whichever citation style I (or those I share it with) choose.

[<img class=" wp-image-880 alignleft" style="border: 1px solid black;margin: 5px" alt="ZoteroAddRecordAddAuthor" src="http://rockarch.org/programs/digital/bitsandbytes/wp-content/uploads/2013/11/ZoteroAddRecordAddAuthor-300x295.png" width="210" height="207" srcset="http://blog.rockarch.org/wp-content/uploads/2013/11/ZoteroAddRecordAddAuthor-300x295.png 300w, http://blog.rockarch.org/wp-content/uploads/2013/11/ZoteroAddRecordAddAuthor-304x300.png 304w, http://blog.rockarch.org/wp-content/uploads/2013/11/ZoteroAddRecordAddAuthor.png 655w" sizes="(max-width: 210px) 100vw, 210px" />](http://rockarch.org/programs/digital/bitsandbytes/wp-content/uploads/2013/11/ZoteroAddRecordAddAuthor.png)The "Abstract" field is a very useful one if you'd like to create an online annotated bibliography. There is also a tab for notes, where you could jot down reading notes or even just the table of contents.

Also note that you can add multiple authors as well as editors and other "creators" (see the "add" button at left.)

But what about those useful annotations? This is one aspect of Zotero that I am still trying to troubleshoot. The only way I've been able to export the references with annotations has been to create a "report" rather than a "bibliography." That is functional, but the drawback is that I have only been able to have it sort the references alphabetically _by title_ rather than by author. In any case, the following is the procedure for creating a report:

Select all references, right click, and choose the option at the bottom, "Generate Report from Items."

[<img class="alignnone  wp-image-893" style="border: 1px solid black" alt="ZoteroStandaloneExportGenerateReportBibliography" src="http://rockarch.org/programs/digital/bitsandbytes/wp-content/uploads/2013/11/ZoteroStandaloneExportGenerateReportBibliography.png" width="1128" height="632" srcset="http://blog.rockarch.org/wp-content/uploads/2013/11/ZoteroStandaloneExportGenerateReportBibliography.png 1128w, http://blog.rockarch.org/wp-content/uploads/2013/11/ZoteroStandaloneExportGenerateReportBibliography-300x168.png 300w, http://blog.rockarch.org/wp-content/uploads/2013/11/ZoteroStandaloneExportGenerateReportBibliography-1024x573.png 1024w, http://blog.rockarch.org/wp-content/uploads/2013/11/ZoteroStandaloneExportGenerateReportBibliography-500x280.png 500w" sizes="(max-width: 1128px) 100vw, 1128px" />](http://rockarch.org/programs/digital/bitsandbytes/wp-content/uploads/2013/11/ZoteroStandaloneExportGenerateReportBibliography.png)

The report will immediately appear as a pdf, which you can save, print, share, etc.

[<img class="alignnone  wp-image-895" style="border: 1px solid black" alt="ZoteroStandaloneGenerateReport" src="http://rockarch.org/programs/digital/bitsandbytes/wp-content/uploads/2013/11/ZoteroStandaloneGenerateReport.png" width="878" height="975" srcset="http://blog.rockarch.org/wp-content/uploads/2013/11/ZoteroStandaloneGenerateReport.png 878w, http://blog.rockarch.org/wp-content/uploads/2013/11/ZoteroStandaloneGenerateReport-270x300.png 270w" sizes="(max-width: 878px) 100vw, 878px" />](http://rockarch.org/programs/digital/bitsandbytes/wp-content/uploads/2013/11/ZoteroStandaloneGenerateReport.png)

Solving the annotation problem is my top priority moving forward with Zotero. I'm checking the forums and the Zotero blog and hope to have it sorted out so that the tool can be as fully useful as possible.

**Collaboration**

The beauty of Zotero is that we can build libraries collaboratively. For now, our RAC group is as small as can be, but I'd be happy to add any interested colleagues. If you'd like to give it a go, let me know and I can invite you to the group.
